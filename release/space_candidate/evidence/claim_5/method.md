# Claim 5 method

The fixed campaign command requests complete recursive Git trees for both
pinned author repositories through the public GitHub API with an explicit
User-Agent. It retains every path, Git blob SHA, and size in a machine-readable
manifest, and separately hashes the notebook containing the published score
numbers.

The audit checks independently for the exact 13-dimensional truths, all four
posterior sets, executable caustics/EPL/SIE/Sérsic model construction, and MALA
code/tuning. Paper-only omissions are recorded separately. A standard-library
checker re-scans the saved Git trees without importing the audit functions.

The negative control supplies plausible path names with zero-byte contents and
invalid hashes, and permutes the top two score labels. The availability checker
must reject the fake files and the ranking checker must reject the permuted
true-model ranking. The verifier succeeds only when the `BLOCKED` status itself
is rigorously established; it would exit nonzero if the recursive release
audit were incomplete or essential assets were actually present.
